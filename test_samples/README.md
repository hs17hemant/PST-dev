# pstwriter smoke-test harness

Self-contained sample inputs + a one-shot script that exercises the
[pstwriter](../README.md) CLI end-to-end. Lives next to the writer (not in
the TM Vault project) because it tests the writer itself.

## Run it

```
test_writer.cmd
```

Just double-click it, or run from cmd / PowerShell. Exit code `0` on full
success, `1` on any failure. Output is also captured by automation:

```powershell
Start-Process .\test_writer.cmd -NoNewWindow -Wait `
    -RedirectStandardOutput out.txt -RedirectStandardError err.txt
```

There is also a `.ps1` version (richer reporting) for users with execution
policy set to allow local scripts.

## What gets tested

| Step | Tool | Expectation |
|---|---|---|
| 1 | `pstwriter_tests.exe --reporter compact` | Catch2 unit suite. 13 known failures are missing optional golden samples in `tests/golden/` — the script flags them as a NOTE only. |
| 2 | `pst_convert mail sample_mail.json out_mail.pst` | Two-message Graph list (text + html bodies, attachment, internet headers). |
| 3 | `pst_info out_mail.pst` | Must print `ALL CHECKS PASSED`. |
| 4 | `pst_convert contacts sample_contacts.json out_contacts.pst` | Two-contact Graph list (multi-email, business + home address, birthday). |
| 5 | `pst_info out_contacts.pst` | Must print `ALL CHECKS PASSED`. |
| 6 | `pst_convert calendar sample_calendar.json out_calendar.pst` | Two-event Graph list (online meeting, all-day, attendees). |
| 7 | `pst_info out_calendar.pst` | Must print `ALL CHECKS PASSED`. |

## Manual Outlook gate

`pst_info` checks structural integrity. The **manual gate** mentioned in
the writer's `KNOWN_UNVERIFIED.md` for M7-M9 is:

1. Open each generated `.pst` in Microsoft Outlook (File -> Open & Export
   -> Open Outlook Data File).
2. Confirm the Inbox / Contacts / Calendar folder shows the items with
   the right subjects / names / start times.
3. Open one item of each type to confirm bodies, recipients, and
   attendees render correctly.

Failures here typically indicate named-property gaps documented in
`KNOWN_UNVERIFIED.md` (M8-1, M9-1).

## Files

| File | Purpose |
|---|---|
| `sample_mail.json`     | Graph `messages` list — 2 messages |
| `sample_contacts.json` | Graph `contacts` list — 2 contacts |
| `sample_calendar.json` | Graph `events` list   — 2 events |
| `test_writer.cmd`      | The harness (pure batch, no execution policy needed) |
| `test_writer.ps1`      | Same harness, richer output, requires unsigned-script execution |
| `out_*.pst`            | Generated by the harness — safe to delete |

## Adding more cases

Drop a new JSON file into this directory and add a `call :run <kind> <json> <pst>`
line in `test_writer.cmd`. Schema is the [Microsoft Graph](https://learn.microsoft.com/en-us/graph/api/resources/message)
`message` / `contact` / `event` resource — the writer accepts a single
object, a bare array, or a `{"value":[...]}` envelope.

## Wiring the writer into TM Vault

This harness only validates the writer in isolation. Wiring it into the
backend's `tm_backend/workers/restore-worker/pst_export.py` (currently
Aspose-based) is a separate task — see the writer README "API at a glance"
for the C++ entry points, or shell out to `pst_convert.exe` from the
worker for a quick cross-language path.
